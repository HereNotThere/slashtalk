Elysia — our version: ^1.3.0

A fast Bun-native HTTP framework with TypeBox schema validation that
doubles as OpenAPI generation.

Load-bearing gotchas
--------------------

- Every route plugin is a factory
      (db, redis?) => new Elysia({ name, prefix })…
  The `name` field is REQUIRED. Elysia dedups plugins by name; missing
  or duplicate names cause silent re-mounts or dropped handlers.
  See docs/design-docs/core-beliefs.md #3.

- .use() chain order matters. Auth plugins must be composed before any
  route that relies on their derived context. See apps/server/src/app.ts.

- Schema validation uses `t.` (TypeBox). Do not mix Zod — we lose the
  OpenAPI/TypeBox bridge.

- Auth split is by route prefix (core-beliefs.md #2):
    /v1/*             → apiKeyAuth only
    /auth/* + /api/*  → jwtAuth only

- CORS + @elysiajs/openapi are mounted first in app.ts. Changing order
  breaks the /openapi route.

Where we use it
---------------

- apps/server/src/app.ts          — plugin composition
- apps/server/src/auth/github.ts  — githubAuth + cliAuth factories
- apps/server/src/ingest/routes.ts
